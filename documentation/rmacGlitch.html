The Rmac Glitch

The Rmac glitch is where on certain custom levels, the stage is beaten immediately. This is thought to be caused by there being triangles with zero area within the StageDef collision.

Fixes

  • Wings 3D: Select all objects, right click, and select "Cleanup".
  • Blender: Select all objects, press W, and select "Remove doubles".

This should remove zero-area faces

Prevention

An issue considered to commonly cause it it using "connect" to triangulate (Pressing C in Wings 3D) instead of the Tessellate > triangulate method. Consider using the latter way for triangulation.
